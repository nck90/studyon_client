# STUDYON 화면 정의서

## 1. 문서 개요

- 제품명: `STUDYON`
- 문서명: `Screen Specification`
- 문서 버전: `v1.0`
- 작성일: `2026-04-14`
- 문서 목적: STUDYON의 핵심 화면 목록과 각 화면의 UI 구성, 상태, 액션, 연결 흐름을 정의한다.

---

## 2. 문서 사용 방법

이 문서는 다음 용도로 사용한다.

- UX/UI 와이어프레임 설계 기준
- 프론트엔드 화면 구현 기준
- QA 테스트 시나리오 작성 기준
- API 연결 우선순위 정의 기준

각 화면은 아래 형식으로 작성한다.

- `화면 ID`
- `화면명`
- `사용자`
- `목적`
- `진입 경로`
- `핵심 컴포넌트`
- `주요 상태`
- `주요 액션`
- `이동/연결 화면`

---

## 4. 학생 앱 화면 정의

## S-01 로그인

- 사용자: `학생`
- 목적: 학생 본인 인증 및 세션 시작
- 진입 경로: 앱 첫 진입, 로그아웃 후 재진입, 자동 로그아웃 후 재진입

### 핵심 컴포넌트

- 로고/서비스명
- 학생 번호 입력 필드
- 이름 입력 필드
- 로그인 버튼
- QR 로그인 버튼
- 자동 로그인 상태 영역
- 오류 메시지 토스트/배너

### 주요 상태

- 기본 상태
- 입력 중 상태
- 로그인 처리 중
- 로그인 실패
- 중복 로그인 감지
- 자동 로그인 가능 상태

### 주요 액션

- 학생 번호 입력
- 이름 입력
- 로그인 요청
- QR 스캔 진입
- 자동 로그인 진행

### 이동/연결 화면

- 성공 시 `S-02 홈`
- QR 인증 성공 시 `S-02 홈`
- 중복 로그인 정책 경고 모달 표시

---

## S-02 홈

- 사용자: `학생`
- 목적: 오늘의 상태와 빠른 행동을 한 화면에서 제공
- 진입 경로: 로그인 완료 후 기본 진입 화면

### 핵심 컴포넌트

- 상단 사용자 인사 영역
- 오늘 상태 요약 카드
- 출결 상태 카드
- 좌석 상태 카드
- 오늘 계획 요약 카드
- 오늘 공부 시간 카드
- 빠른 액션 버튼 영역
- 공지/알림 영역
- 하단 탭바

### 주요 상태

- 미입실 상태
- 입실 완료 / 공부 미시작 상태
- 공부 진행 중 상태
- 일시정지 상태
- 퇴실 완료 상태
- 오늘 계획 없음 상태

### 주요 액션

- 입실
- 퇴실
- 공부 시작
- 계획 추가
- 학습 기록 추가
- 공지 상세 열기

### 이동/연결 화면

- `S-03 출결`
- `S-04 좌석`
- `S-05 오늘의 계획 목록`
- `S-07 공부 세션`
- `S-08 학습 기록 등록`
- `S-09 리포트`

### 와이어프레임 메모

- 상단에 오늘의 핵심 수치 2~3개를 먼저 보여준다.
- 중간 영역은 출결, 좌석, 계획, 공부 상태 순으로 카드 배치한다.
- 하단 CTA는 현재 상태에 따라 가장 필요한 액션 1개를 강조한다.

---

## S-03 출결

- 사용자: `학생`
- 목적: 입실/퇴실 처리와 체류 상태 확인
- 진입 경로: 홈의 출결 카드, 하단 탭 또는 빠른 액션

### 핵심 컴포넌트

- 현재 출결 상태 배지
- 입실 시간 표시
- 퇴실 시간 표시
- 오늘 체류 시간 표시
- 입실 버튼
- 퇴실 버튼
- 출결 이력 리스트

### 주요 상태

- 미입실
- 입실 완료
- 퇴실 완료
- 지각 표시
- 조기 퇴실 표시

### 주요 액션

- 입실 요청
- 퇴실 요청
- 출결 이력 조회

### 이동/연결 화면

- 완료 후 `S-02 홈`
- 출결 이력 상세 모달

### 와이어프레임 메모

- 현재 상태와 CTA를 상단 고정 영역에 둔다.
- 과거 이력은 하단 리스트로 단순 구성한다.

---

## S-04 좌석

- 사용자: `학생`
- 목적: 본인 좌석 확인 및 좌석 상태 조회
- 진입 경로: 홈의 좌석 카드

### 핵심 컴포넌트

- 내 좌석 번호 카드
- 좌석 배치도
- 좌석 상태 범례
- 빈 좌석 리스트
- 좌석 변경 요청 버튼
- 예약/잠금 표시

### 주요 상태

- 지정 좌석 있음
- 지정 좌석 없음
- 좌석 변경 가능
- 좌석 변경 불가
- 빈 좌석 없음

### 주요 액션

- 좌석 배치도 조회
- 빈 좌석 조회
- 좌석 변경 요청

### 이동/연결 화면

- 좌석 변경 요청 모달
- `S-02 홈`

### 와이어프레임 메모

- 상단은 내 좌석 정보, 하단은 배치도 중심으로 배치한다.
- 모바일에서는 배치도와 리스트를 토글할 수 있게 한다.

---

## S-05 오늘의 계획 목록

- 사용자: `학생`
- 목적: 오늘의 계획을 확인하고 관리
- 진입 경로: 홈의 계획 카드, 하단 탭

### 핵심 컴포넌트

- 날짜 헤더
- 계획 요약 수치
- 계획 카드 리스트
- 과목 태그
- 목표 시간 표시
- 우선순위 표시
- 추가 버튼

### 주요 상태

- 계획 없음
- 계획 1개 이상 존재
- 일부 완료
- 전체 완료

### 주요 액션

- 계획 추가
- 계획 상세 열기
- 완료 체크
- 수정 진입
- 삭제

### 이동/연결 화면

- `S-06 계획 등록/수정`
- 계획 상세 바텀시트 또는 모달

### 와이어프레임 메모

- 상단에는 총 계획 개수와 예상 공부 시간을 보여준다.
- 카드형 리스트에서 우선순위가 높은 항목이 먼저 오도록 한다.

---

## S-06 계획 등록/수정

- 사용자: `학생`
- 목적: 당일 학습 계획 생성 및 수정
- 진입 경로: 계획 목록의 추가/수정 버튼

### 핵심 컴포넌트

- 과목 선택 드롭다운
- 세부 학습 내용 입력
- 목표 공부 시간 입력
- 우선순위 선택
- 저장 버튼
- 삭제 버튼

### 주요 상태

- 신규 작성
- 기존 수정
- 필수값 누락
- 저장 중
- 저장 완료

### 주요 액션

- 과목 선택
- 내용 입력
- 시간 입력
- 우선순위 설정
- 저장
- 삭제

### 이동/연결 화면

- 저장 후 `S-05 오늘의 계획 목록`

### 와이어프레임 메모

- 입력 순서는 과목, 내용, 목표 시간, 우선순위로 단순화한다.
- 저장 버튼은 항상 하단 고정 영역에 둔다.

---

## S-07 공부 세션

- 사용자: `학생`
- 목적: 공부 시간 측정 및 세션 상태 제어
- 진입 경로: 홈의 공부 시작 버튼, 하단 탭

### 핵심 컴포넌트

- 대형 타이머
- 현재 세션 상태 배지
- 시작 시간
- 누적 공부 시간
- 누적 휴식 시간
- 시작 / 일시정지 / 재시작 / 종료 버튼
- 현재 연결 계획 표시

### 주요 상태

- 시작 전
- 공부 중
- 일시정지 중
- 종료 완료
- 자동 종료 안내

### 주요 액션

- 공부 시작
- 일시정지
- 재시작
- 종료
- 연결 계획 선택

### 이동/연결 화면

- 종료 후 `S-08 학습 기록 등록`
- 종료 후 `S-02 홈`

### 와이어프레임 메모

- 타이머는 화면 중앙 최상위 계층으로 크게 배치한다.
- 버튼은 상태별로 1~2개만 강조해 혼란을 줄인다.

---

## S-08 학습 기록 등록

- 사용자: `학생`
- 목적: 세션 종료 후 학습량을 입력
- 진입 경로: 공부 세션 종료 후 자동 진입, 홈의 학습 기록 추가 버튼

### 핵심 컴포넌트

- 과목 선택
- 페이지 수 입력
- 문제 수 입력
- 진도율 입력
- 완료 체크
- 메모 입력
- 저장 버튼

### 주요 상태

- 신규 입력
- 세션 연결 상태
- 저장 중
- 저장 완료

### 주요 액션

- 학습량 입력
- 완료 체크
- 메모 저장

### 이동/연결 화면

- 저장 후 `S-09 리포트`
- 저장 후 `S-02 홈`

### 와이어프레임 메모

- 타이핑 부담을 줄이기 위해 숫자 입력은 스텝퍼나 숫자 패드 친화형으로 설계한다.

## S-09 리포트

- 사용자: `학생`
- 목적: 개인 공부 성과 확인
- 진입 경로: 홈, 하단 탭

### 핵심 컴포넌트

- 기간 필터 탭
- 오늘 / 주간 / 월간 공부 시간 카드
- 과목별 분포 차트
- 계획 대비 달성률 카드
- 출석률 카드
- 최근 학습 기록 리스트

### 주요 상태

- 오늘 데이터 없음
- 주간/월간 집계 있음
- 계획 미작성 상태

### 주요 액션

- 기간 변경
- 과목별 상세 보기
- 기록 상세 열기

### 이동/연결 화면

- 학습 기록 상세 모달
- `S-10 랭킹`

### 와이어프레임 메모

- 숫자 KPI 카드와 차트의 비중을 분리한다.
- 첫 화면은 오늘 기준, 상단 탭으로 기간 전환이 적합하다.

---

- 목적: 본인 순위와 경쟁 현황 확인
- 진입 경로: 하단 탭 또는 홈 배너

### 핵심 컴포넌트

- 기간 선택 탭
- 랭킹 유형 탭
- 내 순위 카드
- 전체 랭킹 리스트
- 배지/포인트 요약 카드

### 주요 상태

- 일간 랭킹
- 주간 랭킹
- 월간 랭킹
- 데이터 없음

### 주요 액션

- 기간 변경
- 유형 변경
- 내 위치로 이동

### 이동/연결 화면

- 배지 상세 모달

### 와이어프레임 메모

- 상단에는 내 순위를 고정하고 하단에 전체 리스트를 둔다.
- 개인 동기부여가 핵심이므로 내 위치 강조가 필요하다.

### 주요 상태

- 기본 조회

### 주요 액션

- 로그아웃

### 이동/연결 화면

- 로그아웃 후 `S-01 로그인`

## 5. 관리자 웹 화면 정의

## A-01 로그인

- 사용자: `관리자`, `원장`
- 목적: 관리자 세션 시작
- 진입 경로: 관리자 웹 첫 진입

### 핵심 컴포넌트

- 아이디 입력
- 비밀번호 입력
- 로그인 버튼
- 오류 메시지

### 주요 액션

- 로그인

### 이동/연결 화면

- 관리자 권한이면 `A-02 대시보드`
- 원장 권한이면 `O-01 통합 대시보드`

---

## A-02 대시보드

- 사용자: `관리자`
- 목적: 실시간 자습실 상태를 한눈에 파악
- 진입 경로: 로그인 후 기본 화면

### 핵심 컴포넌트

- 글로벌 필터 바
- KPI 카드 영역
- 좌석 점유 현황 패널
- 미입실 학생 패널
- 공부 미시작 학생 패널
- 비활동 학생 패널
- 빠른 액션 패널

### 주요 상태

- 정상 운영
- 경고 상태 존재
- 데이터 로딩 중

### 주요 액션

- 학생 상세 열기
- 알림 보내기
- 좌석 관리 이동
- 출결 관리 이동

## 10. 최종 정의

STUDYON의 화면 구조는 학생의 당일 행동 흐름과 관리자의 실시간 운영 흐름을 동시에 지원하도록 설계한다.  
학생 화면은 빠른 행동과 기록 중심, 관리자 화면은 모니터링과 제어 중심, TV 화면은 가독성과 전달력 중심으로 정의한다.

이 문서를 기준으로 다음 단계인 DB 설계, API 명세, 실제 와이어프레임 제작으로 이어갈 수 있다.
