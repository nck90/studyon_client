# STUDYON IA

## 1. 문서 개요

- 제품명: `STUDYON`
- 문서명: `Information Architecture`
- 문서 버전: `v1.0`
- 작성일: `2026-04-14`
- 문서 목적: STUDYON의 사용자별 화면 구조, 메뉴 체계, 주요 화면 관계를 정의한다.

---

## 2. IA 설계 원칙

STUDYON의 정보구조는 다음 원칙을 따른다.

### 원칙 1. 사용자 역할별 분리

- 학생
- 관리자
- 원장
- TV 디스플레이

각 역할은 사용하는 정보, 목적, 진입 방식이 다르므로 채널 단위로 분리한다.

### 원칙 2. 학생은 행동 중심, 관리자는 상태 중심

- 학생은 `지금 무엇을 해야 하는가`가 중요하다.
- 관리자는 `지금 어떤 상태인가`가 중요하다.

따라서 학생 앱은 행동 흐름 중심으로, 관리자 앱은 모니터링 중심으로 구성한다.

### 원칙 3. 하루 단위 흐름을 최우선

학생의 핵심 루프는 아래와 같다.

1. 로그인
2. 입실
3. 좌석 확인
4. 계획 작성
5. 공부 시작
6. 학습량 기록
7. 퇴실
8. 리포트 확인

IA는 이 일일 루프를 가장 짧은 클릭 수로 수행하도록 설계한다.

### 원칙 4. 실시간 운영성 우선

관리자와 TV는 정적 정보보다 실시간 정보 비중이 높다.  
따라서 대시보드, 상태 패널, 자동 새로고침성 정보가 상위 구조에 위치한다.

## 3. 채널 구조 요약

STUDYON은 4개의 주요 채널로 구성된다.

1. `학생 앱`
2. `관리자 웹`
3. `원장/상위 관리자 웹`
4. `TV / 공용 디스플레이`

- 출결

## 5-3. 학생 앱 주요 화면 정의

### 1. 로그인

- 목적: 빠르게 학생 식별 및 세션 시작
- 주요 데이터: 학생 번호, 이름, QR 토큰, 기기 상태
- 주요 액션: 로그인, QR 스캔, 자동 로그인

### 2. 홈

- 목적: 오늘 해야 할 행동을 한 화면에 제공
- 주요 데이터:
  - 오늘 출결 상태
  - 좌석 상태
  - 오늘 계획 개수
  - 현재 공부 여부
  - 오늘 누적 공부 시간
- 주요 액션:
  - 입실
  - 퇴실
  - 공부 시작
  - 계획 추가
  - 학습 기록 추가

### 3. 출결

- 목적: 입실/퇴실과 체류 상태를 명확히 관리
- 주요 데이터:
  - 현재 상태
  - 입실 시간
  - 퇴실 시간
  - 오늘 체류 시간
- 주요 액션:
  - 입실
  - 퇴실

### 4. 좌석

- 목적: 내 좌석과 빈 좌석 상태를 확인
- 주요 데이터:
  - 지정 좌석
  - 좌석 상태
  - 빈 좌석 목록
  - 잠금/예약 상태
- 주요 액션:
  - 좌석 조회
  - 좌석 변경 요청

### 5. 오늘의 계획

- 목적: 오늘 공부할 내용을 구조화
- 주요 데이터:
  - 과목
  - 세부 내용
  - 목표 시간
  - 우선순위
  - 완료 상태
- 주요 액션:
  - 생성
  - 수정
  - 삭제
  - 완료 체크

### 6. 공부

- 목적: 실시간 공부 세션 제어
- 주요 데이터:
  - 현재 세션 상태
  - 시작 시각
  - 누적 공부 시간
  - 휴식 시간
- 주요 액션:
  - 시작
  - 일시정지
  - 재시작
  - 종료

### 7. 학습 기록

- 목적: 공부 시간 외 산출량 입력
- 주요 데이터:
  - 과목
  - 페이지 수
  - 문제 수
  - 진도율
  - 메모
- 주요 액션:
  - 기록 추가
  - 수정
  - 완료 체크

### 8. 리포트

- 목적: 본인 성과 확인
- 주요 데이터:
  - 오늘/주간/월간 공부 시간
  - 과목별 비중
  - 출석률
  - 계획 달성률
- 주요 액션:
  - 기간 필터 변경

### 9. 랭킹

- 목적: 동기부여 제공
- 주요 데이터:
  - 기간별 순위
  - 랭킹 유형
  - 배지
  - 포인트
- 주요 액션:
  - 랭킹 타입 변경
  - 기간 변경

## 6-3. 관리자 웹 주요 화면 정의

### 1. 대시보드

- 목적: 운영 핵심 지표를 한 화면에서 확인
- 주요 데이터:
  - 현재 입실 학생 수
  - 좌석 점유율
  - 빈 좌석 수
  - 미입실 학생 수
  - 공부 미시작 학생 수
  - 비활동 학생 수
- 주요 액션:
  - 특정 학생 상세 보기
  - 알림 보내기
  - TV 화면 제어 이동

### 2. 학생 관리

- 목적: 학생 마스터 데이터 관리
- 주요 데이터:
  - 학생 기본 정보
  - 소속 반/그룹/학년
  - 좌석 배정
  - 출결 상태
- 주요 액션:
  - 등록
  - 수정
  - 삭제
  - 좌석 배정

### 3. 좌석 관리

- 목적: 공간 운영 관리
- 주요 데이터:
  - 좌석 번호
  - 상태
  - 배정 학생
  - 예약 여부
  - 잠금 여부
- 주요 액션:
  - 좌석 배정
  - 잠금
  - 예약
  - 상태 변경

### 4. 출결 관리

- 목적: 출석 정책과 일일 출결 운영
- 주요 데이터:
  - 날짜별 출결 현황
  - 지각/조퇴 상태
  - 학생별 출결 이력
- 주요 액션:
  - 출결 조회
  - 예외 처리
  - 통계 다운로드
  - 미출석 알림

### 5. 학습 현황

- 목적: 학생별/반별 학습 상태 분석
- 주요 데이터:
  - 학생별 공부 시간
  - 반별 총 시간
  - 과목별 분포
  - 목표 달성률
- 주요 액션:
  - 기간 필터
  - 학생 상세 이동
  - 반별 비교

### 6. 랭킹 관리

- 목적: 운영 철학에 맞는 경쟁 지표 관리
- 주요 데이터:
  - 기간별 랭킹 목록
  - 랭킹 타입별 점수
  - 가중치
- 주요 액션:
  - 기준 변경
  - 가중치 설정
  - 노출 범위 설정

### 7. 알림 관리

- 목적: 예외 상황 대응 및 공지 운영
- 주요 데이터:
  - 알림 타입
  - 발송 대상
  - 발송 시간
  - 상태
- 주요 액션:
  - 공지 발송
  - 자동 알림 설정
  - 발송 이력 확인

### 8. TV 제어

- 목적: 공용 화면 송출 콘텐츠 제어
- 주요 데이터:
  - 현재 활성 화면
  - 화면별 표시 옵션
  - 문구
- 주요 액션:
  - 화면 전환
  - 텍스트 수정
  - 노출 데이터 선택

원장은 관리자보다 상위 관점의 요약 및 정책 화면이 중요하다.  
원장 웹은 관리자 웹과 별도 메뉴를 두기보다, 상위 메뉴 또는 별도 모드로 제공할 수 있다.

### 권장 1depth 메뉴

- 통합 대시보드
- 운영 리포트
- 성과 분석
- 정책 관리
- 권한 관리

## 8. TV / 공용 디스플레이 IA

### 송출 모드

## 8-1. TV 화면 구조

```text

├─ 랭킹 모드
```

## 9. 공통 객체 기반 IA

화면 구조를 데이터 객체 기준으로 묶으면 아래와 같다.

### 학생 중심 객체

- 학생 프로필
- 출석 기록
- 학습 계획
- 학습 세션
- 학습량 기록
- 리포트
- 랭킹 결과

### 운영 중심 객체

- 좌석

- 반 / 그룹 / 학년
- 알림
- 정책 설정
- TV 송출 설정

즉, IA는 `학생 객체 흐름`과 `운영 객체 흐름`의 두 축으로 설계된다.

## 12. 사용자 흐름 기준 연결 구조

### 학생 주요 플로우

`로그인 → 홈 → 입실 → 좌석 확인 → 계획 작성 → 공부 시작 → 학습 기록 → 퇴실 → 리포트`

### 관리자 주요 플로우

`로그인 → 대시보드 → 학생 상태 확인 → 출결/학습 상세 → 알림 발송`

### 운영자 설정 플로우

`설정 → 출결 정책 / 랭킹 정책 / 권한 정책 → TV 제어 반영`

## 13. 최종 IA 정의

STUDYON의 IA는 단순 메뉴 나열형 구조가 아니라,  
학생의 `하루 학습 루프`와 관리자의 `실시간 운영 루프`를 병렬로 지원하는 구조로 설계한다.

정리하면 다음과 같다.

- 학생 앱은 `행동 중심 IA`
- 관리자/원장 웹은 `상태 및 통제 중심 IA`
- TV는 `읽기 전용 송출 IA`

이 구조를 기준으로 다음 단계인 화면 목록 정의, 와이어프레임, DB 설계, API 설계로 자연스럽게 이어갈 수 있다.
