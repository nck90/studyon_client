STUDYON 테스트 설치 가이드

이 문서는 비전공자도 따라할 수 있게 최대한 쉽게 적은 설치 안내서입니다.

목차
1. 안드로이드에서 APK로 설치해서 테스트하는 방법
2. 아이패드/iPhone에서 TestFlight 내부 테스트로 설치하는 방법

==================================================
1. 안드로이드에서 APK로 설치해서 테스트하는 방법
==================================================

준비물
- 안드로이드 폰 또는 태블릿
- 설치할 APK 파일
- 인터넷 연결

APK가 무엇인가요?
- APK는 안드로이드 앱 설치 파일입니다.
- 컴퓨터의 설치 파일처럼 생각하면 됩니다.

설치 순서

1. APK 파일을 휴대폰으로 받습니다.
- 카카오톡, 이메일, 구글 드라이브, AirDrop 대체 앱, USB 전송 등 아무 방식이나 괜찮습니다.
- 파일 이름 예시는 보통 이런 식입니다.
  - app-debug.apk
  - app-release.apk

2. 휴대폰에서 APK 파일을 엽니다.
- 보통 다운로드 앱 또는 내 파일 앱에서 찾을 수 있습니다.
- 파일을 탭하면 설치 화면이 뜹니다.

3. 처음 설치하는 경우 "알 수 없는 앱 설치" 허용이 필요할 수 있습니다.
- 메시지가 뜨면 "설정"으로 이동합니다.
- 설치를 허용할 앱을 선택합니다.
  - 예: Chrome
  - 예: 내 파일
  - 예: 카카오톡
- "이 출처 허용" 또는 비슷한 문구를 켭니다.

4. 다시 APK 파일로 돌아가서 설치를 누릅니다.
- "설치" 버튼을 누릅니다.
- 잠시 기다립니다.

5. 설치가 끝나면 "열기"를 누릅니다.
- 또는 홈 화면에서 앱 아이콘을 찾아 실행합니다.

문제가 생길 때

- "설치가 차단되었습니다"가 뜨는 경우
  - 알 수 없는 앱 설치 허용이 안 된 상태입니다.
  - 위 3번 단계를 다시 확인합니다.

- "앱이 설치되지 않았습니다"가 뜨는 경우
  - 이미 같은 앱이 다른 서명으로 설치되어 있을 수 있습니다.
  - 기존 앱을 삭제하고 다시 설치해봅니다.

- 저장 공간 부족
  - 사진/동영상/앱을 조금 삭제한 후 다시 시도합니다.

- 설치는 됐는데 실행이 안 되는 경우
  - 앱을 완전히 종료 후 다시 실행합니다.
  - 기기를 재부팅한 뒤 다시 시도합니다.

기존 버전이 있을 때 업데이트 방법
- 새 APK를 다시 설치하면 덮어씌워질 수 있습니다.
- 덮어씌우기 실패 시 기존 앱을 삭제 후 다시 설치합니다.

테스트 후 확인할 것
- 로그인 되는지
- 홈 화면이 열리는지
- 버튼이 눌리는지
- 오류 문구가 뜨는지

오류 제보할 때 보내주면 좋은 것
- 어떤 기기인지
  - 예: 갤럭시 S23
- 안드로이드 버전
- 어느 화면에서 오류가 났는지
- 오류 문구 스크린샷

==================================================
2. 아이패드/iPhone에서 TestFlight 내부 테스트로 설치하는 방법
==================================================

중요
- 내부 테스트는 일반 링크만으로 바로 되는 것이 아닙니다.
- 테스트할 사람의 Apple ID 이메일이 App Store Connect에 내부 테스터로 추가되어 있어야 합니다.

준비물
- iPhone 또는 iPad
- Apple ID
- App Store 접속 가능 상태
- TestFlight 앱
- 초대 메일 또는 내부 테스트 권한

설치 전 준비

1. iPhone/iPad에서 App Store를 엽니다.
2. "TestFlight"를 검색합니다.
3. TestFlight 앱을 설치합니다.

내부 테스트에 초대받는 사람 기준 절차

1. 담당자에게 내 Apple ID 이메일을 전달합니다.
- 이 이메일은 실제로 iPhone/iPad에 로그인된 Apple ID여야 합니다.

2. 담당자가 App Store Connect에서 나를 내부 테스터로 추가합니다.
- 이 단계는 테스터 본인이 하는 것이 아니라 운영자가 합니다.

3. 초대 메일이 오면 메일을 엽니다.
- 보낸 사람은 보통 Apple 또는 App Store Connect 관련 메일입니다.
- 메일 안의 초대 링크를 누릅니다.

4. 링크를 누르면 TestFlight가 열립니다.
- 설치할 앱이 보이면 "설치"를 누릅니다.

5. 설치가 끝나면 TestFlight 안에서 "열기"를 누릅니다.
- 또는 홈 화면에서 앱을 실행합니다.

초대 메일이 안 올 때

- 스팸 메일함 확인
- Apple ID 이메일이 맞는지 다시 확인
- 담당자가 App Store Connect에 실제로 추가했는지 확인
- 초대 수락이 완료되었는지 확인

TestFlight에서 설치가 안 보일 때

- TestFlight 앱을 완전히 종료 후 다시 열기
- Apple ID가 맞는지 확인
- 기기에서 App Store 로그인 계정 확인

업데이트 방법
- 새 테스트 버전이 올라오면 TestFlight 앱 안에서 "업데이트"가 보일 수 있습니다.
- 자동 업데이트를 켜면 새 빌드가 자동으로 설치될 수도 있습니다.

TestFlight에서 자주 보는 상태

- 설치
  - 바로 설치 가능 상태

- 업데이트
  - 새 버전이 올라온 상태

- 만료됨
  - 테스트 기간이 지나 설치할 수 없는 상태

내부 테스트와 외부 테스트 차이

- 내부 테스트
  - App Store Connect 사용자만 가능
  - 심사 없이 빠르게 테스트 가능

- 외부 테스트
  - 일반 사용자 초대 가능
  - Apple TestFlight 심사가 필요할 수 있음

테스트 후 확인할 것
- 로그인 되는지
- 주요 기능이 실행되는지
- 화면이 이상하게 깨지지 않는지
- 버튼 누를 때 멈추지 않는지

오류 제보할 때 보내주면 좋은 것
- 기기 모델
  - 예: iPad Air 5
  - 예: iPhone 15
- iOS/iPadOS 버전
- 어떤 화면에서 문제가 났는지
- 스크린샷
- 가능하면 짧은 동영상
