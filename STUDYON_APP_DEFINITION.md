# STUDYON 앱 정의서

## 1. 서비스 정체성

STUDYON은 **학원/독서실/자습실 운영을 위한 학습 관리 플랫폼**이다.  
단순 출석 앱이 아니라, 다음 4가지를 하나의 시스템으로 통합한 서비스로 정의할 수 있다.

1. **출결 관리 시스템**
2. **좌석 운영 시스템**
3. **학습 시간 및 학습량 추적 시스템**
4. **관리자/TV용 실시간 운영 대시보드**

즉, 이 앱은 학생이 공부를 기록하고, 관리자가 자습실을 통제하며, 공용 화면으로 동기부여까지 제공하는 **자습실 운영 OS 성격의 서비스**다.

---

## 2. 한 줄 정의

> STUDYON은 학생의 입실, 좌석, 공부 시간, 학습 계획, 학습량, 랭킹을 통합 관리하는  
> **학원형 자습실 운영 및 학습 추적 플랫폼**이다.

---

## 3. 핵심 사용자

### 3-1. 학생

학생은 앱에서 아래 흐름을 수행한다.

- 로그인
- 입실 / 퇴실
- 좌석 확인 및 변경 요청
- 오늘의 학습 계획 작성
- 공부 시작 / 종료
- 학습량 기록
- 개인 리포트 확인
- 랭킹 / 배지 확인

학생 앱의 목적은 단순 체크인이 아니라, **스스로 계획하고 기록하며 성취를 확인하도록 만드는 것**이다.

### 3-2. 관리자

관리자는 운영 관점에서 시스템을 사용한다.

- 학생 등록 / 수정 / 삭제
- 반 / 그룹 / 학년 관리
- 좌석 배정
- 출결 관리
- 실시간 자습실 상태 확인
- 공부 시작 안 한 학생 확인
- 비활동 학생 탐지
- 랭킹 규칙 설정
- 알림 발송
- 통계 다운로드

관리자 기능의 목적은 **자습실 운영 자동화와 예외 상황 대응**이다.

### 3-3. 원장

원장은 운영 성과를 본다.

- 출석률
- 자습실 활용률
- 학생별 / 반별 학습 현황
- 운영 효율 리포트
- 집중도 및 위험 학생 분석

즉, 원장 관점에서는 이 앱이 **운영 리포팅 도구**이기도 하다.

### 3-4. TV / 공용 디스플레이

공용 디스플레이는 사용자라기보다 별도 채널이다.

- 실시간 랭킹 송출
- 자습실 현황 표시
- 오늘의 목표 문구 / 챌린지 표시
- 경쟁 / 동기부여 콘텐츠 출력

이 채널은 운영 데이터의 시각화이자 분위기 조성 장치다.

## 4. 제품 카테고리 분석

이 앱은 아래 카테고리가 결합된 **복합형 에듀테크 SaaS**에 가깝다.

### 4-1. 기본 카테고리

- **에듀테크(EduTech)**
- **출결 관리(Attendance Management)**
- **학습 시간 추적(Study Time Tracking)**
- **학습 행동 데이터 관리(Learning Activity Tracking)**
- **시설/좌석 운영(Seat & Space Management)**

### 4-2. 운영 관점 카테고리

- **학원 운영 시스템**
- **독서실 운영 시스템**
- **스터디카페 운영 보조 시스템**
- **실시간 모니터링 대시보드**

### 4-3. 차별화 포인트

보통 출결 앱은 입실/퇴실만 처리한다.  
보통 타이머 앱은 개인 공부 시간만 기록한다.  
보통 학원 관리 앱은 학생 정보와 공지에 집중한다.

STUDYON은 이 셋을 합쳐서 다음을 만든다.

- 출결 데이터
- 좌석 데이터
- 공부 시간 데이터
- 학습량 데이터
- 동기부여 데이터

따라서 이 앱의 핵심 차별점은 **운영 데이터와 학습 데이터를 동시에 다루는 것**이다.

## 5. 문제 정의

이 앱이 해결하려는 문제는 명확하다.

### 학생 측 문제

- 매일 무엇을 공부할지 체계적으로 기록하지 않음
- 공부 시작/종료 시간이 정확히 남지 않음
- 실제 공부 시간과 체류 시간이 분리되지 않음
- 계획 대비 달성 여부를 스스로 확인하기 어려움
- 동기부여가 약함

### 관리자 측 문제

- 누가 왔는지 수기로 확인하거나 여러 시스템을 오가야 함
- 좌석 사용 현황을 실시간으로 파악하기 어려움
- 입실했지만 공부를 시작하지 않은 학생을 놓치기 쉬움
- 출결과 학습량을 연결해 관리하기 어려움
- 일별/주간 운영 통계를 만들기 번거로움

### 원장 측 문제

- 자습실 운영 효율을 숫자로 보기 어려움
- 어떤 학생이 위험군인지 조기에 발견하기 어려움
- 학부모에게 보여줄 근거 데이터가 부족함

## 6. 핵심 가치 제안

STUDYON의 핵심 가치 제안은 다음과 같다.

1. **자동화된 출결 및 좌석 운영**
2. **학생 중심의 계획-실행-기록 루프 제공**
3. **공부 시간과 학습량의 정량화**
4. **관리자용 실시간 모니터링**
5. **랭킹과 배지를 통한 동기부여**
6. **향후 AI 분석과 학부모 연동이 가능한 데이터 구조 확보**

## 7. 기능 도메인 구조

기능을 보면 이 앱은 크게 6개 도메인으로 나뉜다.

### 7-1. 인증 도메인

- 학생 번호 / 이름 로그인
- QR 로그인
- 좌석 태블릿 자동 로그인
- 세션 유지
- 자동 로그아웃
- 중복 로그인 방지

이 도메인은 학생 식별과 디바이스 식별이 모두 중요하다.

### 7-2. 출결 도메인

- 입실 / 퇴실
- 입실 시간 자동 기록
- 퇴실 시간 자동 기록
- 일별 출석 기록 저장
- 지각 / 조기 퇴실 체크
- 총 체류 시간 계산

이 도메인은 학원 운영의 기본 데이터가 된다.

### 7-3. 좌석 도메인

- 지정 좌석 배정
- 좌석 상태 표시
- 빈 좌석 조회
- 좌석 변경 요청
- 좌석 잠금 / 예약

이 도메인은 실제 자습실 공간 운영과 직접 연결된다.

### 7-4. 학습 실행 도메인

- 오늘 목표 입력
- 과목 선택
- 세부 학습 내용 입력
- 목표 공부 시간 설정
- 우선순위 설정
- 공부 시작 / 종료
- 일시정지 / 재시작
- 휴식 시간 분리 기록

이 도메인은 학생의 당일 행동을 구조화한다.

### 7-5. 학습 기록/분석 도메인

- 과목별 공부 시간
- 페이지 수 / 문제 수 / 진도율
- 완료 체크
- 메모
- 오늘 / 주간 / 월간 리포트
- 계획 대비 달성률
- 출석률

이 도메인은 학생 피드백과 관리자 분석의 중심이다.

### 7-6. 운영/동기부여 도메인

- 랭킹
- 포인트 / 배지
- TV 송출
- 공지 / 알림
- 미입실 / 비활동 / 목표 미달성 알림

이 도메인은 운영 개입과 동기부여를 담당한다.

## 8. 이 앱의 본질적인 사용자 흐름

이 서비스는 아래 흐름으로 이해하면 가장 명확하다.

### 학생 일일 흐름

1. 로그인한다.
2. 입실한다.
3. 좌석을 확인하거나 배정받는다.
4. 오늘의 학습 계획을 입력한다.
5. 공부를 시작한다.
6. 휴식과 공부 시간을 분리 기록한다.
7. 공부한 과목/분량을 남긴다.
8. 퇴실한다.
9. 개인 리포트와 랭킹을 확인한다.

### 관리자 운영 흐름

1. 학생과 좌석을 등록한다.
2. 실시간 입실/미입실 현황을 본다.
3. 공부 시작 안 한 학생, 비활동 학생을 확인한다.
4. 필요 시 알림을 보낸다.
5. 출결/학습 통계를 조회한다.
6. 랭킹 및 TV 화면을 제어한다.
